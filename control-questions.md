# Ответы на контрольные вопросы
## 1. Какая текущая версия MacOS?
MacOS 26 Tahoe
## 2. На основе какой UNIX-подобной системы построена операционная система macOS?
Операционная система MacOS построена на основе Darwin
## 3. К какому типу относится ядро macOS? В чем отличия между XNU и Mach?
Ядро macOS (XNU) относится к гибридному типу (hybrid kernel). XNU - полное ядро операционной системы, а Mach - его компонент
## 4. Какой тип архитектуры у процессоров линейки M1 и выше, производимых Apple?
Процессоры линейки Apple M1, M2, M3 и выше имеют архитектуру ARM
## 5. Что такое виртуализация? Приведите примеры гипервизоров.
Виртуализация — это технология, которая позволяет создавать виртуальные (программные) версии компьютерных ресурсов: операционных систем, серверов, хранилищ данных, сетей и т.д.
Примеры гипервизоров:
1. VMWare ESXi
2. Microsoft Hyper-V
3. Oracle VirtualBox
## 6. Что такое «командная оболочка» / «командный интерпретатор»?
Командная оболочка - это программа, которая обеспечивает интерфейс для взаимодействия пользователя с операционной системой. Она применяет команды, которые пользователь вводит с клавиатуры, и передает их ядру ОС для выполнения.
## 7. Как называется командный интерпретатор и какая его версия в используемой операционной системе?
GNU bash, версия 5.2.21(1)-release (x86_64-pc-linux-gnu)
## 8. Для каких задач предназначено приложение Terminal в macOS?
Основные задачи терминала в macOS:
1. Навигация и управление файлами
2. Настройка системы
3. Установка и запуск программ
4. Работа с Git
5. Автоматизация задач
6. Удаленное администрирование
7. Разработка ПО
## 9. Приведите полный путь к домашней папке пользователя, под которым вы авторизованы в системе. Привести пример команды, позволяющей определить путь к домашнему (текущему) каталогу в консоли.
/home/hhlbby
Полный путь к домашней папке пользователя можно получить с помощью команды ```pwd```
## 10. Какой командой можем вывести содержимое каталога системных конфигурационных файлов?
``` bash
ls -la /etc
```
## 11. Приведите синтаксис команды для установки файлового менеджера Midnight Commander из консоли в macOS. Привести три варианта команды установки из консоли приложения Terminal в macOS.
В macOS нет единого встроенного менеджера пакетов. Midnight Commander устанавливается через сторонние менеджеры пакетов.
``` bash
brew install mc
```
``` bash
sudo port install mc
```
``` bash
curl -O http://ftp.midnight-commander.org/mc-4.8.31.tar.gz
tar -xzf mc-4.8.31.tar.gz
cd mc-4.8.31
./configure && make && sudo make install
```

## 12. Как можно вывести текст руководства (справочной информации) для какой- нибудь команды в консоли?
Для вывода справки используются следующие команды:
-  ```man```
``` bash
man ls
```
- ```--help```
```bash
ls --help
```
## 13. Для каких целей используется команда open?
Команда ```open``` позволяет открывать папки, файлы и приложения
## 14. Приведите примеры команд группировки и перенаправления, используемых в командном интерпретаторе zsh.
- ```>```- перенаправить вывод команды в файл:
  ``` bash
  ls -la > list.txt
  ```
- ```>>``` - перенаправить вывод команды в файл:
  ``` bash
  echo "Новая строка" >> list.txt
  ```
- ```<``` - взять содержимое файла как ввод для команды:
  ``` bash
  wc -l < list.txt
  ```
- ```|``` - группировка: передать вывод одной команды на ввод другой:
  ``` bash
  ls -la | grep ".txt" | wc -l
  ```
## 15. С помощью какой команды, можем изменить командный интерпретатор с zsh на bash?
``` bash
  chsh -s /bin/bash
```
## 16. В каких случаях выполняется инициализация репозитория? Какой командой?
Создание репозитория выполняется, когда есть локальная папка с проектом, и разработчик хочет начать отслеживать изменения в ней
``` bash
  git init
```
## 17. Что такое коммит?
Коммит - это снимок состояния проекта на определенный момент времени.
## 18. Почему перед созданием новой ветки важно создать коммит?
Потому что Git создает новую ветку от текущего состояния
## 19. В каких состояниях может находиться файл в репозитории? Как происходит изменение состояния файла?
В Git файл может находиться в трёх разных состояниях:
1. Modified(Изменен). Для перехода к нему необходимо отредактировать файл
2. Staged (Подготовлен). Для перехода к нему нужно ввести команду ``` git add <файл>```
4. Committed. Для перехода к нему необходимо ввести команду ```git commit -m "Message"```
## 20. Что такое ветка?
Ветка - отдельная линия разработки. Ветки позволяют разработчикам работать над новыми функциями или исправлением ошибок, не мешая основному коду.
## 21. Какое условие является обязательным для формирования ветки master после клонирования репозитория или его инициализации?
Обязательным условием является наличие хотя бы одного коммита
## 22. Что такое HEAD?
```HEAD``` - это указатель, который указывает на текущий коммит или ветку, с которой разработчик работает в рабочей директории
## 23. Способы создания веток.
- ```git branch <имя-ветки>```
- ```git branch <имя-ветки> <хеш-коммита>``` - ветка от конкретного коммита
- ```git checkout -b <имя ветки>``` - создание ветки и мгновенный переход на неё
## 24. Как узнать текущую ветку?
- ```git branch```
- ```git status```
- ```git rev-parse --abbrev-ref HEAD```
## 25. Как переключаться между ветками?
- ```git checkout <имя-ветки>```
- ```git switch <имя-ветки>```
## 26. С помощью какой команды можно слить две ветки?
Перейти на ветку, к которой хотим влить другую ветку и пишем команду ```git merge <имя-ветки-которую-вливаем>```
## 27. Как подключить к инициализированному (созданному) локальному репозиторию внешний репозиторий? Какой командой?
```git remote add origin <ссылка-на-внешний-репозиторий>```
## 28. Какая команда позволяет отменить последний коммит?
```git reset```
## 29. Для чего предназначена команда rebase?
Она предназначена для переноса цепочки коммитов из одной ветки на другой базовый коммит
## 30. В каких случаях требуется создание форка? Приведите способы создания форка, включая консольную команду.
Случаи, когда требуется создать форк:
- Если разработчик не является collaborator'ом в проекте (нет прав на запись в оригинальный репозиторий)
- Разработчик хочет поэкспериментировать, не влияя на основной проект
- Разработчик планирует внести свой вклад в open-source и потом отправить автору через Pull Request
Способы создания форка:
- Через веб-интерфейс
- Консольная команда: ``` bash
git clone <ссылка-на-репозиторий>
cd <имя-репозитория>
git remote add upstream https://github.com/ОРИГИНАЛ_ВЛАДЕЛЕЦ/репозиторий.git
```
